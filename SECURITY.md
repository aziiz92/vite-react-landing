# Security Policy

If you discover a security issue, please **do not** open a public issue with details.

## Reporting

- Preferred: open a **private security advisory** on GitHub (if enabled for the repo).
- Otherwise: email the maintainer with details and a repro (replace this with a real address): `security@example.com`.

## What to include

- A clear description of the issue and impact
- Steps to reproduce (or a PoC)
- Any suggested fix or mitigation
